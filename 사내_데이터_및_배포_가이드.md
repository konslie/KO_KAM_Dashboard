# KAM 대시보드 사내 데이터 전환 및 배포 가이드

본 문서는 개발된 **KAM 대시보드**를 사내망 환경에 배포하고, 실제 사내 데이터(CSV)로 전환하여 구동하기 위한 가이드입니다.

---

## 1. 사내 데이터(CSV) 준비 및 전환 조건

사내 데이터 파일을 적용할 때 아래의 필수 컬럼명과 데이터 형식이 유지되어야 오류 없이 동작합니다.

### 1.1 필수 컬럼 정의 및 데이터 타입
| 컬럼명 | 설명 | 예시 / 데이터 타입 |
| :--- | :--- | :--- |
| **YYYYMM** | 조회 년월 | `202401` (정수형) |
| **TIER** | 고객 Tier 분류 | `Tier 1`, `핵심`, `일반` 등 (문자열) |
| **RVRT_DIV_NM** | 담당 본부/부서명 | `기업영업본부` (문자열) |
| **RVRT_TEAM_NM** | 소속 팀명 | `영업1팀` (문자열) |
| **RVRT_EMP_NM** | 담당 사원명 | `홍길동` (문자열) |
| **PFLS_NM** | 상품명 | `초고속인터넷`, `M2M` (문자열) |
| **SLIN_TOTAL** | 매출액 | `150000000` (소수점/쉼표 없는 숫자형) |
| **EPM_CUST_ID** | 고객 고유 ID | `C12345` (문자열 또는 숫자형) |

### 1.2 파일 이름 설정
*   기본적으로 대시보드는 **`synthetic_b2b_dashboard_20k.csv`** 파일을 읽도록 설정되어 있습니다.
*   **방법 A**: 사내 데이터 파일명을 `synthetic_b2b_dashboard_20k.csv`로 변경하여 덮어쓰기 합니다.
*   **방법 B**: `data.py` 파일의 상단에 정의된 함수에서 로드할 파일명을 수정합니다.
    ```python
    # data.py
    def load_data(filepath="사내_데이터_파일명.csv"):
        ...
    ```

### 1.3 Tier 분류 매핑 (`data.py` 확인 필수)
현재 대시보드는 KA/GA를 구분할 때 특정 Tier 값을 기준으로 동작합니다.
*   **KA (Key Account)**: `Tier 1`, `Tier 2`, `Tier 3`
*   **GA (General Account)**: `핵심`, `W.seg`, `일반`

만약 사내 데이터의 `TIER` 값 이름이 다를 경우, `data.py` 상단에서 아래 코드를 수정해야 합니다.
```python
# data.py 상단
TIER_ORDER = ['Tier 1', 'Tier 2', 'Tier 3', '핵심', 'W.seg', '일반']  # 원하는 Tier 목록으로 수정

# load_data 함수 내부
ka_tiers = ['Tier 1', 'Tier 2', 'Tier 3']  # 사내 KA 기준
ga_tiers = ['W.seg', '핵심', '일반']      # 사내 GA 기준
```

## 2. 폐쇄망(인터넷 차단) 환경을 위한 설정

대시보드 화면(`index.html`)은 기본적으로 로컬의 **`static/js/plotly.min.js`** 파일을 참조합니다.

이미 프로젝트 폴더 내부 `static/js/` 폴더에 Plotly.js 파일이 생성되어 있으므로 별도의 다운로드 없이 바로 오프라인에서 구동할 수 있습니다. 

### (참고) Python 내장 Plotly.js 파일 직접 생성하기
혹시 파일이 누락되었거나 다시 생성해야 한다면, `plotly`가 설치된 콘솔에서 아래 명령어를 실행하여 로컬에 바로 생성할 수 있습니다.
```bash
mkdir -p static/js
python3 -c "import plotly.offline; open('static/js/plotly.min.js', 'w', encoding='utf-8').write(plotly.offline.get_plotlyjs())"
```

## 3. 실행 및 배포 가이드

사내 서버 혹은 개발 PC에서 구동하기 위한 환경 구축 단계입니다.

### 3.1 필수 라이브러리 설치
프로젝트 루트 폴더에서 콘솔을 열고 `requirements.txt`에 정의된 라이브러리들을 설치합니다.
```bash
pip install -r requirements.txt
```
> **주요 패키지**: `Flask`, `pandas`, `plotly`

### 3.2 Flask 서버 구동
서버 구동 파일인 `app.py`를 실행합니다.
```bash
python app.py
```
*   기본 구동 포트는 `5001`번으로 설정되어 있습니다. (예: `http://127.0.0.1:5001`)
*   포트 충돌이 발생할 경우 `app.py` 파일의 맨 마지막 줄에서 포트 번호를 변경할 수 있습니다.
    ```python
    if __name__ == '__main__':
        app.run(debug=True, port=8080)  # 원하는 포트로 변경 가능
    ```
